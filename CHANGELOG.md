# Changelog

以下仅保留当前版本仍成立的用户侧变更与能力说明。

## 2026-06-11 - 1.8.3-bernadette.9

### 公益发布保护

- 新版本改用 QQ 农场公益产品许可证 1.0，明确禁止收费转售、付费代部署、营利性托管及移除公益标识。
- 核心健康接口、Web 控制页和桌面壳统一展示“公益产品”，并增加标识完整性状态。
- 源码迁入私有仓库，公开仓库仅保留签名保护分支及发布包；保护副本不携带构建器和混淆器依赖。
- 旧版本已经授予的 GPL-3.0 权利不追溯撤销。

## 2026-06-11 - 1.8.3-bernadette.8

### 收获追踪与意见入口

- 自家分批催熟后短时快速追收；好友逐块催熟后开启定向追踪并优先复访触发好友。
- 普通周期结束后自动恢复，兼顾收获成功率和运行流畅度。
- 桌面壳及 Web 项目说明新增微信意见群二维码，用于收集功能建议和兼容性反馈。
- 新增收获追踪调度回归测试。

## 2026-06-11 - 1.8.3-bernadette.5

### 好友逐块催熟补扫

- 好友列表已提示可偷、但进场瞬间状态尚未同步时，会等待至少 `500 ms` 并在场内复扫一次。
- 这种瞬时状态差异不再误加默认 `5` 分钟访问冷却，下一轮好友偷菜仍可继续命中后续催熟作物。
- 对好友列表已标记可操作的候选人读取详细地块，以便普通偷菜模式也能动态识别四格作物锚点；不会扩大为全好友详细扫描。

## 2026-06-11 - 1.8.3-bernadette.4

### 四格作物偷菜修复

- 琉璃宝荷等四格作物在好友列表正常显示可偷后，进入农场会把四个覆盖格统一归并到唯一锚点。
- 锚点根据当前好友农场的地块 ID、节点路径和网格坐标动态解析，不依赖 `1/2/5/6` 或主锚点 `5` 等固定位置。
- 定点收获不再依次点击四个覆盖 landId，而是只调用锚点植物的成熟特效，失败时回退单地收获协议。
- 多格作物成熟后不再强依赖锚点地块的 `canSteal` 标记，可按成熟状态进入定点收获。
- 新增实际布局与任意位置布局回归测试，覆盖锚点路径缺失、`canSteal` 缺失等情况。

## 2026-06-11 - 1.8.3-bernadette.3

### 低延迟与成功率平衡

- 好友偷菜默认周期恢复为 `90` 秒，继续使用轻量状态读取，避免用降低命中率换取流畅度。
- 自动种植默认检查周期缩短为 `15` 秒；成功收获后会立即把补种任务置为到期，并在串行调度的下一轮优先补种。
- 后端串行调度缓冲调整为 `750 ms`，在降低空闲等待的同时继续避免任务并发抢占游戏运行时。

## 2026-06-11 - 1.8.3-bernadette.2

### 运行时性能优化

- 降低 Web 控制台默认轮询频率，健康检查改为 `5` 秒，自动农场状态改为 `4` 秒。
- 自动农场忙碌时，控制台会自动放慢健康检查、日志状态、奖励弹窗、土地详情和账户状态刷新，减少与游戏窗口抢资源。
- CDP 预览不再在注入成功后自动启动，避免实时截图流占用运行时；需要看画面时可手动打开预览。
- 好友偷菜默认走轻量状态读取；仅启用作物黑名单时才读取完整地块列表，减少进好友农场后的卡顿。
- 默认好友偷菜间隔调整为 `120` 秒，进好友等待为 `1000 ms`，动作等待为 `500 ms`，后端串行调度缓冲为 `1000 ms`。

## 2026-05-12

### 作物铲除协议直发与批量支持

- 新增 `/api/shovel-lands-batch` 与 `gameCtl.shovelLandsBatch`，可按 `landIds` 多地块直接派发 `REQUEST_ERASE_PLANT` 协议，不再依赖打开地块详情、点击铲除按钮或确认弹窗。
- 土地详情页单个地块“铲除”按钮已切换到协议直发链路：仍保持单按钮操作体验，但底层走批量接口传入单个 `landId`，避免铲除时弹出游戏 UI。
- 批量铲除会在执行前过滤空地、非主地块的 2x2 作物占用格和不可用地块，并在执行后逐地块校验 `growing -> empty` 结果。
- 已实测协议直发批量铲除成功：地块 `9`、`10` 一次调用均从 `growing / hasPlant=true` 变为 `empty / hasPlant=false`。

### 单地块铲除作物真实实现

- 土地详情单个地块卡片已新增“铲除”操作，并与无机/有机施肥按钮放在同一操作区；仅自己农场、非空地块可用，2x2 作物会提示在主地块操作。
- 新增 `/api/shovel-land` 后端入口与 `gameCtl.shovelLand` 运行时方法，支持按 `landId` 指定单个作物铲除，并在操作完成后刷新土地详情状态。
- QQ lite bundle 已接入真实铲除链路：打开目标地块详情、点击游戏原生 `land_detail/erase`、在 `performErasing` 弹出确认后继续点击确认按钮，不再停留在空壳 UI。
- 修复“点击铲除按钮无反应”的问题：补齐网关新路由重启后的调用链，并修正旧交互面板/旧 manager 导致目标地块详情未重新打开的时序问题。
- 已实测铲除成功：地块 `7` 的“石竹花”从 `growing / hasPlant=true` 变为 `empty / hasPlant=false`，空地数量从 `8` 增加到 `9`。

### 地块变异识别与图片显示修复

- 地块详情新增变异状态展示：会按当前地块运行时 `activeTypes` 显示变异类型、图标、效果说明或变异作物预测，并在地块汇总中统计变异数量。
- 变异类型判定已改为使用运行时 `activeTypes` 对照表：`1=冰冻`、`2=爱心`、`3=暗化`、`4=湿润`、`5=黄金`、`6=哈哈`、`7=塔塔`，不再用图鉴序号或变异作物名称反推类型。
- 变异作物图片现在由后端统一匹配并生成稳定 URL；前端不再二次拼接变异图片，也不再追加动态时间戳，修复变异图片反复闪烁的问题。
- 新增变异图鉴资源读取与图片接口，支持普通、黄金及组合变异作物按阶段显示对应图片；效果型变异会保留源作物图片并显示倍率效果。
- 清理无用映射副本与 CSV 对照文件，保留运行时仍需要的 JSON 映射，减少资源目录噪音。

### 自动施肥批量催熟与连续催熟

- 自动施肥已改为复用自动种植的多地块批量思路：按同一批候选地块生成施肥计划，并通过游戏协议批量直发，减少逐地块操作带来的等待与误判。
- 修复种植后立即施肥时的状态标记问题：施肥季节标识已改为稳定的 `地块:作物:季节`，不再因为成熟时间变化把同一批作物误判成新对象。
- 种植后的新地块会主动清理上一轮施肥标记，避免刚种完就被旧作物状态挡住无机或有机施肥。
- 催熟联动收获已从无限等待式轮询改为明确的催熟/收获闭环；默认仍保持单轮催熟，任务完成后会正常结束。
- 新增“连续催熟”开关：开启后，如果一轮催熟后作物仍未成熟且仍在催熟阈值内，会继续催熟并尝试收获，直到成熟收获、无可催熟目标或无真实进展时结束。
- 保留“单轮最大地块数”配置项，`0` 继续表示全部地块；“地块间隔”设置已从前端移除，后端仍强制按 `0` 处理以兼容旧版本配置。
- 项目全局版本号已统一更新为 `V1.8.2`；页面标题、包信息与 README 中的版本展示现已同步。

## 2026-05-10

### QQ 启动自动补丁与运行时入口重做

- QQ 路线启动时会自动构建并检测当前最新补丁；如果目标 `game.js` 未打补丁或仍是旧补丁，会在启动前自动写入新补丁。
- 自动补丁失败时不会阻断主程序启动，会回退到手动补丁方式，并提示可在运行时弹窗中继续使用“一键打补丁”处理。
- 未显式配置 QQ `game.js` 或 AppID 时，自动补丁会尝试按最近活跃的 QQ 小程序缓存目录定位目标，减少首次启动时手动填写路径的步骤。
- Web 控制台的运行时页已改为弹出式面板，集中展示 QQ 宿主、bundle hash、补丁目标、CDP 与注入状态。
- 左侧常驻工具栏新增“运行时”图标入口，并与开始/停止自动化、GitHub 图标竖向对齐；顶部运行时页签不再作为主要入口显示。

### 好友帮忙次数持久化修复

- 修复好友帮忙次数只更新在内存、未正确写入本地缓存的问题；自动帮忙结束后会持久化到 `data/friend-help-exp-cache.json`，重启后今日累计次数可继续沿用。

## 2026-05-09

### 仓库刷新控制与物品页签

- 仓库页新增手动刷新控制之外的后台刷新设置：可开启/关闭定时刷新，并自定义刷新间隔，默认仍为 `5` 分钟。
- 新增“仅自动出售时刷新”开关；启用后会自动停用前面的定时刷新，只在到达预设自动出售时间时执行仓库刷新与出售检查。
- 仓库物品展示新增与游戏内一致的页签：`果实 / 超变果实 / 种子 / 道具`，并在页签上显示当前分类数量。
- 修复普通果实被误识别为超变果实的问题：现在只按游戏仓库来源页签判断“超变果实”，不再把运行时 `mutation` 数值直接当作分类依据。

### 异地登录重连与异常日志

- 系统配置新增 `异地登录重连` 独立开关，默认关闭；检测到“账号已在其他地方登录”弹窗后，会按配置等待后点击重新登录，默认等待 `5` 分钟。
- 该能力独立于白屏重启和进程守护，不会因为异地登录弹窗触发小游戏重启流程。
- 异地登录重连已接入异常日志：检测到弹窗时记录 `账户在别处登陆，等待xx重连`，点击重新登录成功后记录 `账户在别处登陆，重连成功`。
- 微信/CDP 路线已改为运行时主动事件上报：通过 `Runtime.addBinding` 注入事件桥，识别到弹窗或重连成功时直接通知网关写入异常日志，不再由网关轮询检测。
- QQ WS 路线同步接入宿主事件上报，`qq-host` 会把运行时的异地登录重连事件转发给网关并写入同一类异常日志。

### 好友偷菜区域拆分与冷却可配置

- 好友偷菜已从通用调度里独立成单独区域，前端现在可以集中配置偷菜开关、巡查间隔、每轮好友数、作物黑名单策略与黑名单列表。
- 偷菜冷却已开放为可调参数：普通冷却与黑名单跳过冷却都可在前端直接设置，填 `0` 即表示不启用对应冷却；默认值仍保持原来的 `5` 分钟与 `10` 分钟。
- 后端好友巡查的冷却判定已改为读取配置值，不再写死固定时长，状态页也会同步展示当前生效的冷却配置。

### 日志页筛选、清空与异常独立持久化

- 自动农场日志页已补齐筛选与操作增强：支持 `全部日志 / 自己农场事件 / 好友农场事件 / 异常事件` 视图，并新增 `清空日志` 按钮，便于快速清理普通巡检历史后继续观察当前轮次。
- 修复“无待施肥地块”误判为异常的问题；像 `自动施肥：成功 0 块，跳过 N 块，失败 0 块` 这类结果现在会按正常跳过显示，不再进入异常统计。
- 修复好友帮忙判断日志误判为异常的问题；`浇水/除草/除虫 N → N` 这类帮忙无进展记录现在按普通信息日志显示，不再进入异常事件筛选或异常历史。
- 异常日志现已独立落盘到 `data/task-abnormal-events.ndjson`；异常筛选不再依赖普通任务日志最近若干条结果，后续普通日志再多也不会把历史异常事件挤掉。
- 普通任务日志清空后，会同步清掉内存里的近期任务事件，但独立异常日志会保留，方便后续继续追溯异常链路。
- 修复自动施肥异常文案中 `plant interaction manager not found` 这类报错的业务翻译：通知与日志现会统一归类为 `目标化肥容器不足`，同时保持异常级别不变。

### Web 控制台悬浮按钮与 Lite 客户端布局微调

- Web 控制台左侧新增全局悬浮 `开始 / 停止自动化` 共用按钮，并与 GitHub 入口同组悬浮显示；运行中会自动切换为停止态，空闲时显示开始态。
- Lite 客户端主卡片底部留白已上调，底边与“运行信息 / 今日统计”区域之间保留更明显的安全间距，避免底部视觉过挤。

### macOS Lite 停止服务修复

- 修复 macOS 环境下 Lite 客户端点击“停止服务”后无法真正停止的问题：停止流程现已支持通过端口反查 PID，并在 macOS 上用进程信号回收服务进程。

### 自动施肥批量协议直发与间隔配置

- 自动施肥批量流程已改成真正按多个 `landId` 走游戏协议处理，不再使用“补跑”或单地块兜底重试；一次批次只会跑一轮。
- 批量施肥会在同一批次内按地块顺序直发协议；地块间隔由后端固定为 `0`，不再在前端开放配置。
- 施肥批次的运行日志已同步收口为批量直发口径，便于区分真实批量执行与单地块操作。
- 施肥日志现已按策略区分“智能无机”和“种植无机”，避免 `种植无机，催熟有机（旧策略）` 被误显示成智能无机轮。

### 自动施肥配置保存修复

- 修复自动施肥配置保存失败的问题，前端保存时会正确写入并校验施肥相关参数。

## 2026-05-08

### 启动入口收敛与 Windows Lite 命令行隐藏

- 启动入口已收敛为两个文件：`Windows_start.bat` 和 `Mac_start.sh`，不再在启动时询问操作系统，平台由对应启动器自动注入。

- 自己农场自动收获现已把 `待清理枯萎 / eraseDead` 纳入收获任务链路，并复用一键收获 `HARVEST` 执行清理；日志会单独显示为 `一键清理枯萎`，避免识别到枯萎地块却一直提示“无待处理项”。

### 运行内存、注入卡顿与奖励弹窗拦截优化

- QQ 注入包默认切换为 `lite` 模式：新增 `button-lite.js` 和 `npm run qq:build-lite`，默认 bundle 体积控制在 210KB 内，减少 QQ 小程序注入后的解析与常驻诊断开销；需要完整诊断能力时仍可通过 `FARM_QQ_BUNDLE_MODE=full` 或 `--bundle-mode full` 使用完整 `button.js`。
- `button.js` 默认不再自动启动 runtime spies、interaction spies 和 reconnect watcher；相关诊断能力保留为显式调用入口，避免空闲状态持续监听运行时消息。
- 网关侧减少内存峰值：任务日志改为尾部读取，自动农场 `lastResult` 改为摘要，预览管理不再常驻大体积 base64 帧。
- 小程序卡顿优化已覆盖 QQ 与微信路线：自动重连预检查改为短时间窗复用，控制台高频只读 `gameCtl.*` 调用增加短 TTL 缓存，并在健康状态里暴露统一 `rpcStats`，用于继续定位慢调用。
- 奖励弹窗拦截开关不再限制 QQ WS，微信/CDP 路线也复用同一套 `gameCtl.getRewardPopupInterceptorState()` 与 `gameCtl.setRewardPopupInterceptorEnabled()`。
- 修复奖励弹窗拦截关闭后仍无法恢复的问题：强制隐藏节点前会记录原始状态，关闭开关或停止期间会恢复被改过的节点；首次启用时也会立即扫描，不再等下一轮间隔。
- 奖励弹窗拦截配置新增持久化文件 `data/reward-popup-interceptor-state.json`；控制页面关闭后再打开会显示保存状态，状态接口会把保存配置同步到当前 QQ/微信 runtime，避免读到旧 runtime 临时状态。

### 消息推送重开收口、异常聚合与恢复通知

- 修复消息推送重新开启后，关闭期间积累的异常日志被一次性补发的问题；现在重新开启异常通知时，会把日志观察游标直接对齐到当前末尾，并清空旧的超时累计与观察态，不再回放 backlog。
- 异常通知新增“短窗口聚合”策略：同类异常首条仍会立即推送，后续 1 分钟内重复出现的同指纹异常不再逐条刷屏，而是合并成一条摘要通知，提示这段时间内又重复出现了多少次。
- 连接异常、运行时异常和超时阈值告警现已接入恢复通知；当检测到连接恢复或上下文重新就绪后，会额外推送一条 `农场异常恢复`，包含恢复时间、持续时长和累计出现次数。
- 异常通知的观察开关已进一步收口到 `异常通知` 本身：仅关闭异常推送而保留日志监控配置时，不再继续暗中累计旧异常状态，避免后续重新开启时立即触发陈旧告警。

### 自动施肥单轮上限、催熟联动收口与收获实验移除

- 自动施肥新增“单轮最大地块数”配置：`0` 表示不限制；大于 `0` 时，会在一次 `own_fertilizer` 任务内对所有施肥策略、所有 phase 以及催熟联动循环共用同一额度，达到上限后立即结束本轮施肥，不再继续后续施肥阶段。
- `催熟联动收获` 的触发范围已收口为“仅催熟轮触发一次自动收获”；非催熟施肥阶段不再额外拉起自动收获，避免把普通补肥轮误当成连续收获循环的一部分。
- 原先的“并发收获（实验性）”已整体移除：后端并发调度分支、配置持久化、状态输出以及控制页开关均已下线，不再保留这条实验链路。
- 修复自动施肥在超时恢复阶段可能出现“日志显示成功施肥，但实际未生效”的误判问题；现在只有在复查里明确观察到成熟时间缩短时，才会把该地块记为施肥成功。
